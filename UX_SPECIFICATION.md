# UX-СПЕЦИФИКАЦИЯ: РАЗГОВОРНЫЙ ИНТЕЛЛЕКТ БОТА
## AI Brand Studio — как бот должен общаться

---

## ФИЛОСОФИЯ

Бот — это не анкета и не меню. Это **цифровой бренд-стратег**, который ведёт клиента через процесс. Он слушает, понимает контекст, адаптируется под человека и никогда не заставляет делать то, что человеку неудобно.

**Три правила:**
1. Бот всегда понимает что хочет человек, даже если тот выражается неточно
2. Бот никогда не игнорирует сказанное — он отвечает на то, что написано, а не на то, что ожидал
3. Бот предлагает альтернативы, а не настаивает на единственном пути

---

## ГЛОБАЛЬНЫЕ ПРИНЦИПЫ UX

### Бот понимает смысл, а не ключевые слова
Intent router отправляет каждое сообщение в Claude API для классификации. Claude понимает русский, английский, опечатки, сленг, длинные и короткие формулировки. Таблицы сценариев в этом документе — ПРИМЕРЫ для иллюстрации, а не исчерпывающий список. Пользователь может выразить то же намерение тысячей разных способов — бот поймёт.

### Кнопки = подсказки, не ограничения
Кнопки показывают рекомендуемые действия. Но текстом можно сделать всё то же самое и даже больше. Кнопки — это помощь, а не клетка.

### Опасные действия = подтверждение кнопкой
Удаление, перезапуск, сброс — всегда через подтверждение. Текстом можно инициировать, но подтвердить — только кнопкой.

### Контекст не теряется
Если пользователь в середине брифинга спросил "покажи мои проекты" — бот показывает проекты, но помнит что брифинг активен и предлагает вернуться.

### Тон
Дружелюбный, уверенный, краткий. Не канцелярит. Не панибратство. Как общается хороший менеджер проектов в студии — профессионально, но по-человечески.

### Бот никогда не молчит
На любое сообщение — ответ. Если не понял — переспрашивает. Если не может выполнить — объясняет почему и предлагает альтернативу.

## СЦЕНАРИИ ПО ЭТАПАМ

### ЭТАП 0: ПЕРВЫЙ КОНТАКТ (после /start, нет проектов)

**Что показывает бот:**
```
Привет! Я — AI бренд-студия Maks Martin.

Помогу создать ваш бренд. Для этого я задам пару вопросов, 
и на основе ваших ответов придумаю идею и стратегию, 
затем название и визуальный стиль.

Как начнём?

[🚀 Новый проект]  [❓ Как это работает]
```

**При нажатии "Как это работает":**
```
Вы получите готовый бренд — всё что нужно чтобы запуститься 
и выглядеть профессионально:

✦ Стратегия — кто вы, для кого, чем отличаетесь от конкурентов
✦ Название — варианты имени для бизнеса + слоган
✦ Голос бренда — как говорить с клиентами, какой тон и стиль
✦ Визуальный стиль — цвета, шрифты, настроение, направление дизайна
✦ Готовый документ — всё собрано в один файл, который можно 
  передать дизайнеру, подрядчику или использовать самому

Как это происходит — 6 коротких шагов:
1. Знакомство — расскажете о бизнесе (или загрузите файл)
2. Стратегия — предложу идею и позиционирование
3. Название — несколько вариантов на выбор
4. Концепции — 3-5 творческих направлений
5. Визуал — цвета, шрифты, общий стиль
6. Финальный документ — всё в одном месте

Каждый шаг можно обсудить и доработать. 
Обычно занимает 30-40 минут.

[🚀 Начать]
```

**Сценарии свободного текста:**

Таблица ниже показывает ПРИМЕРЫ — не полный список. Бот использует Claude API для классификации намерений, поэтому понимает любую формулировку на русском и английском, включая опечатки, сленг и нестандартные фразы. Intent router определяет СМЫСЛ сообщения, а не ищет точные слова.

| Пользователь пишет (примеры) | Бот понимает (intent) | Бот отвечает |
|---|---|---|
| "Хочу сделать логотип для кофейни" / "Нужен бренд" / "Запускаю бизнес" | NEW_PROJECT | "Отлично! Назовём проект — например, «Кофейня» или любое рабочее название." |
| "У меня есть бриф" / "Могу скинуть файл" / "Есть презентация" | UPLOAD + нет проекта | "Конечно! Создадим проект и загрузите файл — я всё изучу. Как назовём проект?" |
| "Что ты умеешь?" / "Как работает?" / "Зачем это нужно?" / "А это долго?" | QUESTION | Отвечает на конкретный вопрос по-человечески. НЕ выдаёт список модулей. Например: "Я помогу создать бренд с нуля — от идеи до готового документа. Обычно это 30-40 минут." |
| "Покажи мои проекты" / "Что у меня есть?" / "Мои проекты" | LIST_PROJECTS | "У вас пока нет проектов. Начнём? [🚀 Новый проект]" |
| Отправляет файл (PDF, документ) | UPLOAD + нет проекта | "Вижу файл! Создадим проект чтобы обработать. Как назовём?" |
| "Привет" / "Здравствуйте" / Эмодзи / Стикер | GREETING | "Привет! Готовы создать бренд? [🚀 Новый проект]" |
| Любое непонятное сообщение | UNCLEAR | "Я — бот для создания брендов. Хотите начать новый проект? [🚀 Новый проект] [❓ Как это работает]" |

### ЭТАП 1: СОЗДАНИЕ ПРОЕКТА (awaiting_input = "project_name")

**Что показывает бот:**
```
Как назовём проект?
Можно название компании, продукта или просто рабочее название.
```

**Сценарии:**

| Пользователь пишет | Бот понимает | Бот отвечает |
|---|---|---|
| "Sunrise" | PROJECT_NAME | "Проект «Sunrise» создан! Начинаем?" + кнопки [💬 Брифинг] [📎 Загрузить файл] |
| "Я ещё не придумал название" | NOT_READY | "Ничего страшного! Можно использовать рабочее название и изменить позже. Например, просто опишите суть — «кофейня», «стартап», «девелопер»." |
| "Покажи проекты" | LIST_PROJECTS (прерывание) | Показывает проекты. Внизу: "Или создайте новый — просто напишите название." |
| "Можно без названия?" | QUESTION | "Название нужно чтобы отличать проекты. Напишите любое рабочее — потом его можно изменить." |
| "Забей, не хочу" | CANCEL | "Хорошо! Когда будете готовы — просто напишите. [🚀 Новый проект]" |

### ЭТАП 2: СТАРТ РАБОТЫ (проект создан, брифинг не начат)

**Что показывает бот после создания проекта:**
```
Проект «Sunrise» создан!

Как хотите начать?

[💬 Расскажу о проекте]  [📎 Загрузить файл с описанием]
```

**Важно:** НЕ начинать брифинг автоматически! Дать выбор. Многие клиенты приходят с готовым брифом.

**Сценарии свободного текста (примеры — бот понимает любые формулировки):**

| Примеры сообщений | Intent | Бот отвечает |
|---|---|---|
| "Это кофейня в центре Москвы" / "Мы строим дома" | CONTINUE_DIALOG | Принимает как начало рассказа, запускает брифинг с этим контекстом |
| "У меня есть презентация" / "Могу скинуть PDF" / "Вот файл" | UPLOAD | "Загрузите — я изучу и соберу информацию сам." |
| "А что дальше будет?" / "Что мне нужно рассказать?" | QUESTION | "Я задам несколько вопросов о вашем бизнесе: чем занимаетесь, для кого, чем отличаетесь. Это займёт минут 10. Или загрузите файл — я разберусь сам." |

### ЭТАП 3: БРИФИНГ (awaiting_input = "briefing")

**Ключевой принцип:** бот ведёт диалог, но в любой момент человек может прервать, спросить, загрузить файл или попросить пропустить вопрос. Бот адаптируется, а не настаивает.

**Сценарии (примеры формулировок — бот понимает смысл, не конкретные слова):**

| Примеры сообщений | Intent | Бот отвечает |
|---|---|---|
| Любой ответ по теме вопроса | CONTINUE_DIALOG | Продолжает брифинг как обычно |
| "У меня есть файл" / "Можно документ скинуть?" / "Лучше пришлю бриф" | UPLOAD_INSTEAD | "Отлично! Загрузите — я извлеку информацию. Не придётся отвечать на вопросы, которые уже раскрыты в документе." |
| "Пропусти" / "Не знаю" / "Потом отвечу" | SKIP | "Хорошо, идём дальше. [следующий вопрос]" |
| "Что ты уже знаешь?" / "Покажи собранное" | SHOW_PROGRESS | "Вот что я уже понял о проекте: [краткая сводка]. Продолжим?" |
| "Сколько ещё?" / "Долго ещё?" | QUESTION | "Осталось примерно X вопросов. Можете загрузить файл — я дособеру сам. Или скажите «хватит» — соберу из того что есть." |
| "Хватит" / "Достаточно" / "Давай дальше" / "Закончи брифинг" | FORCE_COMPLETE | "Понял! Собираю бриф из того, что есть. [📋 Посмотреть бриф] [✅ Всё верно, продолжаем]" |
| "Удали проект" / "Не хочу продолжать" / "Отмена" | DELETE/CANCEL | Подтверждение кнопкой |
| "Мои проекты" / "Покажи проекты" | LIST_PROJECTS | Показывает список + "Вы в проекте «Sunrise» (знакомство). [↩️ Вернуться]" |
| "Переключись на [название]" | SWITCH_PROJECT | Переключает. "Теперь в проекте «Дом». [▶️ Продолжить]" |

### ЭТАП 4: РЕВИЗИЯ МОДУЛЯ (awaiting_input = "*_revision")

**Сценарии:**

| Пользователь пишет | Бот понимает | Бот отвечает |
|---|---|---|
| Пишет правки | CONTINUE_DIALOG | Применяет ревизию как обычно |
| "Всё ок, идём дальше" | APPROVE | Утверждает и переходит к следующему модулю |
| "Начни этот модуль заново" | RESTART_MODULE | "Пересоздать [модуль]? Текущие результаты будут заменены. [🔄 Да, заново] [❌ Оставить как есть]" |
| "Вернись на шаг назад" | GO_BACK | Возвращает к предыдущему модулю |

### ЭТАП 5: МЕЖДУ МОДУЛЯМИ (модуль завершён, ждём решения)

**Сценарии:**

| Пользователь пишет | Бот понимает | Бот отвечает |
|---|---|---|
| "Дальше" / "Продолжай" | NEXT_MODULE | Запускает следующий модуль |
| "Покажи результат ещё раз" | SHOW_RESULT | Показывает результат текущего модуля |
| "Перейди к неймингу" | GO_TO_MODULE(3) | Если можно — переходит. Если нет: "Для нейминга нужна готовая бренд-платформа. Давайте сначала завершим модуль 2." |
| "Скачай всё что есть" | EXPORT | Генерирует и отправляет текущие результаты |

### УНИВЕРСАЛЬНЫЕ (работают ВСЕГДА, на любом этапе)

| Пользователь пишет | Бот отвечает |
|---|---|
| "Помощь" / "Что делать" | Краткое описание текущего этапа + что можно делать |
| "Статус" | Текущий проект, этап, прогресс |
| "Мои проекты" | Список проектов + где сейчас находишься |
| "Удали проект [название]" | Подтверждение кнопкой |
| "Удали все проекты" | Подтверждение кнопкой с предупреждением |
| "Начни заново" | Подтверждение: что именно заново — весь проект или текущий модуль |

## ОБРАБОТКА ФАЙЛОВ НА ЛЮБОМ ЭТАПЕ

| Ситуация | Бот отвечает |
|---|---|
| Файл до создания проекта | "Вижу файл! Создадим проект чтобы обработать. Как назовём?" |
| Файл во время брифинга | "Получил! Изучаю... [анализ]. Дополнил бриф информацией из файла. Продолжим?" |
| Файл после брифинга | "Получил! Использую как дополнительный контекст для следующих этапов." |
| Файл не PDF | "Я работаю с PDF-файлами. Можете конвертировать или просто рассказать текстом." |

## ФОРМАТ ОТВЕТОВ БОТА

### Правила текста:
- Максимум 3-4 предложения перед кнопками
- Никаких списков модулей если не спросили
- Никаких технических терминов (не "awaiting_input", а "жду ваш ответ")
- Эмодзи — умеренно, только функциональные (📎 для файлов, ✅ для подтверждения)

### Правила кнопок:
- Максимум 3 кнопки в ряд
- Текст кнопки — глагол в императиве: "Загрузить", "Продолжить", "Удалить"
- Главное действие — первая кнопка
- Отмена/возврат — последняя кнопка
- Опасные действия — красный эмодзи (⚠️)

### Правила подтверждений:
- Удаление: "Удалить проект «X»? Это нельзя отменить. [⚠️ Удалить] [❌ Отмена]"
- Сброс: "Начать модуль заново? Текущие результаты будут заменены. [🔄 Заново] [❌ Оставить]"
- Переключение проекта: без подтверждения (не опасное действие)

## АНТИПАТТЕРНЫ (ЧТО БОТ НЕ ДОЛЖЕН ДЕЛАТЬ)

- ❌ Показывать список модулей когда не спрашивали
- ❌ Молчать на сообщение
- ❌ Создавать проект из текста вопроса
- ❌ Начинать брифинг без согласия пользователя
- ❌ Игнорировать файл и продолжать задавать вопросы
- ❌ Удалять что-либо без подтверждения кнопкой
- ❌ Показывать кнопки предыдущего этапа
- ❌ Отвечать шаблонно когда можно ответить по контексту
- ❌ Использовать технический язык ("модуль 2", "callback")
- ❌ Заставлять проходить этапы строго по порядку если есть файл
