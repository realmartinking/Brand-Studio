# Brand Research

Скилл для сбора и верификации данных о брендах перед созданием файлов в Brand OS/Brands/.

## Главное правило

**НИКОГДА не выдумывай факты о бренде.** Если не уверен — ставь `status: unverified` и пиши что именно требует проверки. Ложная информация хуже отсутствующей.

---

## Два режима работы

### Режим 1: С верифицированными данными (приоритетный)

Пользователь предоставляет файл `brand-briefs.md` или описание бренда в чате. В этом случае:

1. Бери ТОЛЬКО факты из предоставленного описания
2. Не дополняй из своих знаний без пометки
3. Координаты по 18 осям выводи из предоставленных данных
4. Если данных недостаточно для определения координаты — ставь 0.5 и пометку `# estimated`

### Режим 2: Из собственных знаний (только для глобальных брендов)

Допускается ТОЛЬКО для брендов с мировой известностью (Apple, Nike, Hermès, Chanel, IKEA и т.д.). В этом случае:

1. Используй только общеизвестные факты
2. Ставь `status: verified-from-training` в YAML
3. Не выдумывай детали о владельцах, адресах, ценах

### Запрет

Для нишевых, локальных, региональных брендов (рестораны, локальные марки, барбершопы, небольшие девелоперы) — **НИКОГДА не создавай файл из собственных знаний**. Только из предоставленных пользователем данных или из файла brand-briefs.md.

---

## Классификация бренда перед созданием

Перед созданием файла определи категорию:

**Категория A — Глобальный известный** (Apple, Porsche, Chanel, IKEA, Noma, Aman)
→ Можно из собственных знаний, status: verified-from-training

**Категория B — Известный в индустрии** (Forma, Брусника, 12storeez, Linear, Byredo)
→ Только базовые факты из собственных знаний + обязательная пометка что требует проверки деталей

**Категория C — Нишевый / локальный** (Maya, Pims, Foam, Björk, Endorfin)
→ ТОЛЬКО из предоставленных данных. Без данных — не создавай файл.

---

## Структура верифицированного описания (brand brief)

Минимальный набор данных для создания файла:

```
### [Имя бренда]
- **Что:** [продукт/услуга]
- **Кто владеет:** [владелец/основатель/группа]
- **Где:** [город, страна]
- **Сегмент:** [economy/comfort/business/premium/luxury]
- **Индустрия:** [real-estate/horeca/fashion/tech/beauty/other]
- **Позиционирование:** [1-2 предложения — суть бренда]
- **Характер:** [3-5 ключевых слов]
- **Visual markers:** [что видно глазами — интерьер, упаковка, сайт, лого]
- **Verbal markers:** [как говорит — тон, примеры фраз]
- **Антипод:** [что этот бренд точно НЕ]
```

Если какое-то поле отсутствует — файл всё равно можно создать, но соответствующая координата ставится как `# estimated` и `status: partial`.

## Правила определения координат

### Из чего выводить координаты:

| Данные из brief | Какие оси определяет |
|---|---|
| Сегмент (economy→luxury) | all_few, distance, speed |
| Позиционирование | self_community, roots_horizon, body_mind |
| Характер | loud_quiet, serious_playful, control_release |
| Visual markers | geometry, contrast, density, temperature, weight |
| Verbal markers | distance, serious_playful, loud_quiet |
| Антипод | инверсия — антипод на 0.2, бренд на 0.8 |

### Чего НЕ делать:

- Не ставить координату по названию бренда (Stone ≠ heavy)
- Не ставить координату по стране (Россия ≠ warm, Скандинавия ≠ cool)
- Не копировать координаты ближайшего аттрактора — бренд всегда отличается
- Не ставить все координаты близко к 0.5 — это значит «ничего не определено»

---

## Определение nearest_attractor

1. Прочитай файлы в Brand OS/Attractors/
2. Для каждого аттрактора посчитай разницу по ключевым осям (первые 8 narrative + 6 sensory)
3. Выбери аттрактор с минимальной суммарной разницей
4. Если разница > 0.3 по 5+ осям — отметь как attractor gap, предложи новый аттрактор

---

## Формат выходного файла

Следуй шаблону из Brand OS/_templates/brand-template.md. Дополнительно:

- `status:` в YAML — одно из: `verified`, `verified-from-training`, `partial`, `unverified`
- `source:` в YAML — откуда данные: `user-provided`, `brand-briefs.md`, `training-data`
- Все wikilinks на реальные файлы в Attractors/ и Brands/
- Никаких заглушек [[brand-1]]

---

## Self-check перед сохранением файла

Перед записью файла проверь:

1. ✅ Владелец/основатель — факт, не выдумка?
2. ✅ Сегмент соответствует реальной ценовой позиции?
3. ✅ Все координаты обоснованы данными, а не ассоциациями?
4. ✅ nearest_attractor — реальный файл в Attractors/?
5. ✅ Wikilinks ведут на существующие файлы?
6. ✅ Нет заглушек [[brand-1]]?
7. ✅ status и source заполнены?

Если хоть один пункт не проходит — не сохраняй, исправь.
